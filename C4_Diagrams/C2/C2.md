# Container (C2) Level: Smart City Living Lab

## Overview
This folder provides a structural breakdown of the "black box" defined in the C1 layer. It exposes the **Microkernel (Plug-in) Architecture** by mapping each major deployable unit (Microservice/Container), database, and middleware component. 

## Diagram Details (`C2.puml`)

### User Interfaces
*   **Unified Dashboard App (Flutter):** Provides the single-codebase frontend for web administrators and mobile residents.

### Gateways & Middleware
*   **API Gateway (FastAPI):** Central router for handling external UI connections and enforcing base authentication.
*   **Semantic Middleware (OneM2M):** A dedicated translation container taking raw hardware inputs and standardizing them into universal objects before they hit the internal bus.
*   **Privacy Scrubbing API (FastAPI + Pandas):** A strict boundary pipeline dropping PII before transmitting datasets to Researchers.

### Core Bus & Databases
*   **Central Message Broker (RabbitMQ):** The AMQP microkernel routing all asynchronous telemetry logic across the city.
*   **Telemetry Store (InfluxDB):** The high-write, 30-day Time-Series Database holding raw metrics.
*   **Config & RBAC Store (PostgreSQL):** The safe relational database holding structured user permissions.

### Domain-Specific Engines (The Plugins)
*   **CAM Engine (FastAPI):** High-speed logic answering entrance authentication requests.
*   **EHS Engine (FastAPI):** Runs ML algorithms to forecast air/water trends.
*   **Energy Engine (FastAPI):** Balances AC load and predicts solar output.
*   **Alerting Engine (FastAPI):** Subscribes to critical triggers to dispatch Twilio/SendGrid updates.

## PlantUML Implementation
The structural mapping of exactly *how* these specific technologies communicate (HTTPS vs AMQP vs pure TCP) is rendered in the accompanied `C2.puml` file inside this directory.

# System Context (C1) Level: Smart City Living Lab

## Overview
This folder contains the highest-level description of the Smart City ecosystem. The C1 level treats the entire software system as a single black box, focusing strictly on its boundaries, the stakeholders who interact with it, and the external hardware/services it interfaces with.

## Diagram Details (`C1.puml`)

### Target System
*   **Smart City Living Lab:** The primary centralized software resolving telemetry inputs, running domain logic, and serving dashboard user interfaces.

### External Actors
*   **Campus Managers (EHS, Energy, CAM):** Key administrators who monitor system vitals and establish automation rules.
*   **Residents (End-Users):** General campus population accessing public views and receiving safety alerts.
*   **Researchers (Data Scientists):** External developers extracting 30-day anonymized historical trends.

### External Systems
*   **300 Heterogeneous IoT Nodes:** The physical street-level hardware capturing the data.
*   **Twilio Gateway:** An external REST API used exclusively for dispatching SMS alerts to offline residents.
*   **SendGrid Gateway:** An external REST API used for dispatching high-reliability email reports and alerts.
